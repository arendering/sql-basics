===================
=== 4. Индексы  ===
===================
  Первичный ключ
  ---> Первичный ключ - поля в таблице, позволяющие однозначно идентифицировать запись.
Такой ключ в таблице может быть только один

атрибут PRIMARY KEY - база не даст добавить дубликат.
атрибут AUTO_INCREMENT - база сама увеличивать значение поля

атрибут UNIQUE KEY - создает уникальный индекс по колонке. Т.е в такой колонке могут быть только уникальные значения.
пример:
 CREATE TABLE users (
 ...
 passport VARCHAR(10) NOT NULL,
 UNIQUEY KEY passport (passport) // UNIQUE KEY <index_name> (<column_name_1>, <column_name_2>, ...)
 ...
);
  Создаём уникльный индекс с именем passport по колонке passport

 Если индекс создан по нескольким полям, то уникальность должна быть для всех колонок вместе.

 Индекс
 ---> Индекс (не уникальный, а для быстрого поиска в БД) - для ускорения поиска, фильтрации. Подобно картотекам хранят информацию о том,
где находится та, или иная запись.

 Создать индекс для уже существующей таблицы:
 CREATE INDEX <index_name> ON <table_name>(<column_name>)

 Создать индекс при создании таблицы:
 CREATE TABLE <table_name> (
   ...
   mark VARCHAR(50) NOT NULL,
   INDEX <index_name> (mark)
   ...
 );

 Имеет смысл создавать индексы по колонкам, которые часто используются в блоках WHERE или ORDER BY


 Составной индекс - для ускорения поиска по нескольким стобцам.
 ...
 INDEX main_search(user_id, price) // перечисляем имена столбцов в скобках
 ...

 ---> Добавление и удаление индексов
 можно сделать UNIQUE INDEX - индекс, стобцы которого д.б уникальными на таблицу.

 удалить индекс - DROP INDEX <index_name> ON <table_name>;

 Обыкновенные индексы лучше не использовать на колнках типа TEXT. Для этого есть текстовые индексы.
